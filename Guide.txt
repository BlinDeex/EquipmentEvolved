Equipment Evolved: Stat System Cheat Sheet


1. The Core Architecture
EquipmentStat (The Logic): These are singleton classes (one exists per mod). They define how a stat modifies the player and how it formats its tooltip. They do not store player data.

StatPlayer (The Ledger): This is the ModPlayer. It acts purely as a ledger (a dictionary). It remembers how many points of a stat a player has, and where those points came from (StatSource).


2. How to Create a New Stat
To add a brand new stat to the game (e.g., Fishing Power), you just create a new class that inherits from EquipmentStat.

Step 1: Create the class in Assets/Core/Stats/Definitions/...
Step 2: Override FormatTooltip to tell the UI how to display it.
Step 3: Override the relevant Terraria hook to actually apply the math.

public class FishingPowerStat : EquipmentStat
{
    // 1. How it looks on a Charm or Sealed weapon
    public override string FormatTooltip(float totalValue)
    {
        return $"+{(int)totalValue} Fishing Power"; 
    }

    // 2. How it affects the player
    public override void UpdateEquips(Player player, float totalValue)
    {
        player.fishingSkill += (int)totalValue;
    }
}


3. How to Apply a Stat to a Player
Whenever an Accessory, Armor Piece, Prefix, or Buff wants to give the player stats, you simply add it to the ledger using AddStat.

StatPlayer statPlayer = player.GetModPlayer<StatPlayer>();

Adding a flat stat (e.g., +15 Fishing Power from an Accessory)
statPlayer.AddStat(ModContent.GetInstance<FishingPowerStat>(), 15f, StatSource.Accessory);

Adding a percentage multiplier (e.g., +10% Damage from a Charm)
Note: Pass 0.10f for 10%. The stat class handles the 1f + totalValue math!
statPlayer.AddStat(ModContent.GetInstance<DamageStat>(), 0.10f, StatSource.Charm);

Adding a negative/reduction stat (e.g., -15% Pickaxe Speed)
Note: Pass -0.15f. The stat class will safely resolve 1f + (-0.15f) to 0.85x speed!
statPlayer.AddStat(ModContent.GetInstance<PickSpeedStat>(), -0.15f, StatSource.AccessoryReforge);


4. StatSources & Multiplying
Always tag your stats with the correct StatSource (e.g., StatSource.Charm, StatSource.AccessoryReforge, StatSource.Chestplate).
If you want an armor set to boost all Accessory Reforges by 25%, call this in UpdateEquips:

Increases all stats tagged as AccessoryReforge by 1.25x
statPlayer.MultiplySourceStats(StatSource.AccessoryReforge, 1.25f);

5. How to Read a Stat manually
Usually, StatPlayer automatically passes the totals down to the hooks. But if you need to manually check a stat's value inside a custom weapon or projectile:
StatPlayer statPlayer = player.GetModPlayer<StatPlayer>();

Get the total of ALL sources combined
float totalFish = statPlayer.GetTotalStat(ModContent.GetInstance<FishingPowerStat>());

Get the total from ONE specific source
float charmFish = statPlayer.GetStatFromSource(ModContent.GetInstance<FishingPowerStat>(), StatSource.Charm);


6. How to add your new stat to the RNG Pools
If you want your brand new stat to be rollable on Charms, Sealed Weapons, or Chaotic Weapons, you just add it to their respective lists!

To add to Charms:
Open CharmBalance.cs and add a line to the _statDefinitions lazy-load property:
{ ModContent.GetInstance<FishingPowerStat>(), (CharmRarity.Common, 5f, 15f) },

To add to Sealed Weapons:
Open SealedRollManager.cs and add a line to the _rollPool lazy-load property:
new() { Stat = ModContent.GetInstance<FishingPowerStat>(), MinValue = 5f, MaxValue = 20f, Weight = 1.0f },

To add to Chaotic Weapons:
Open ChaoticRollPool.cs and add a line to the _availableStats lazy-load property:
new() { Stat = ModContent.GetInstance<FishingPowerStat>(), MinValue = 5f, MaxValue = 30f },
